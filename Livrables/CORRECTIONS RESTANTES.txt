Performance et Données (Statistics.js & AdminPanel.js)
L'application télécharge toutes les réservations (getAllReservations) à chaque montage de composant majeur.
Pour une mairie de 20k habitants, si le fichier Sheets atteint 2000-3000 lignes, l'application ralentira.
Optimisation prévue : Je proposerai un système de cache simple ou une vérification pour ne pas rappeler l'API Google si les données ont été chargées il y a moins de x secondes.

Faille de Sécurité & Maintenance : Le mot de passe administrateur (R3sa@M0rep@s78) est codé en dur dans 3 fichiers différents. Si vous voulez le changer, vous devez recompiler l'application.

Surcharge API : Chaque composant recharge toutes les réservations à chaque montage. Avec plusieurs utilisateurs simultanés, vous allez atteindre les quotas de l'API Google Sheets (Read Quota).

Logique "Dimanche" fragile : Le patch dans SingleRoomGrid pour sauter le dimanche provoque un "clignotement" et des recalculs inutiles.

Conflits CSS/JS : L'usage intensif de !important dans le CSS est une conséquence directe de styles inline appliqués via le JS (notamment dans les grilles).
